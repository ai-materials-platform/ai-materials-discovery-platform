# AI Materials Discovery Platform

AI 기반 계산 재료 탐색(Materials Discovery) 및 물성 예측(Property Prediction)을 위한 연구성 프로젝트입니다.

## 프로젝트 목적
- 노트북 중심으로 빠르게 실험을 반복합니다.
- 검증된 로직은 `src/`로 분리해 재사용성을 높입니다.
- 학습 결과와 산출물을 체계적으로 관리합니다.

## 저장소 구조

```text
ai-materials-discovery-platform/
├── notebooks/      # 실험/분석용 Jupyter 노트북
├── data/           # 데이터
├── src/            # 재사용 가능한 Python 모듈
├── models/         # 학습된 모델 및 체크포인트
├── outputs/        # 시각화, 로그, 예측 결과물
├── scripts/        # 학습/추론/유틸 실행 스크립트
└── docs/           # 프로젝트 문서
```

## 권장 워크플로우
1. `notebooks/`에서 아이디어를 빠르게 검증합니다.
2. 반복 사용되는 코드를 `src/`로 이동해 모듈화합니다.
3. 모델 학습 결과를 `models/`에 저장합니다.
4. 그래프/리포트/예측값을 `outputs/`에 정리합니다.
5. 사용 방법과 의사결정을 `docs/`에 문서화합니다.

## 참고
- 빈 디렉터리 유지를 위해 각 폴더에 `.gitkeep` 파일이 포함되어 있습니다.

# AI 소재 발굴 플랫폼 (AI Materials Discovery Platform) v2.0

본 플랫폼은 스테인리스강 소재의 화학 조성과 공정 변수를 기반으로 주요 기계적 물성을 예측하고 분석하는 인공지능 시스템입니다. 이전 버전에서 확장되어 강도뿐만 아니라 연성 지표까지 포함한 종합적인 분석을 제공합니다.

## 🚀 주요 신규 기능 (v2.0 업데이트)

### 1. 4대 핵심 물성 통합 예측
기존의 강도 지표에 재료의 변형 능력을 나타내는 연성 지표를 추가하여 총 4가지 물성을 동시에 예측합니다.
*   **강도(Strength):** 0.2% 항복강도 (Yield Stress), 인장강도 (UTS)
*   **연성(Ductility):** 연신율 (Elongation), 단면 수축률 (Area Reduction)

### 2. 지능형 학습 제어 시스템
*   **사용자 정의 반복 횟수(Iterations):** 데이터의 특성에 따라 학습의 깊이를 직접 조절할 수 있습니다 (기본 2,000회).
*   **조기 종료 (Early Stopping):** 학습 중 더 이상 성능 향상이 없으면 AI가 스스로 판단하여 학습을 중단함으로써 과적합(Overfitting)을 방지하고 모델을 보호합니다.

### 3. 고도화된 데이터 시각화
*   **이중 Y축 예측 그래프:** 단위가 다른 강도(MPa)와 연성(%)을 한 그래프에서 명확하게 비교할 수 있는 이중 축 시스템을 도입했습니다.
*   **상세 성능 분석 (Parity Plot):** 실제값(Actual)과 예측값(Predicted)의 분포를 대각선(y=x) 지표와 함께 제공하여 모델의 정확도를 직관적으로 검증합니다.
*   **반응형 상관관계 히트맵:** 창 크기에 따라 수치 표시 여부가 자동으로 조절되어, 작은 화면에서는 깔끔하게, 큰 화면에서는 상세하게 데이터를 분석할 수 있습니다.

## 🛠 시스템 아키텍처

### Backend Engine
*   **DataEngine:** `STMECH_AUS_SS.xls` 데이터를 로드하고 4개 타겟 변수에 대한 전처리를 수행합니다.
*   **ModelEngine:** Random Forest, GBM, MLP, TFP(확률론적 앙상블) 등 4가지 모델을 지원하며 다중 출력(Multi-output) 예측에 최적화되어 있습니다.

### Frontend GUI
*   **PyQt6 기반 인터페이스:** 모든 메뉴와 안내 문구를 한국어로 최적화하였으며, '맑은 고딕' 글꼴을 적용하여 가독성을 높였습니다.
*   **실시간 학습 피드백:** 학습 진행 상태 및 신뢰도(R²) 점수를 실시간으로 제공합니다.

## 📋 사용 방법
1.  **데이터 선택:** 학습용 엑셀 파일을 선택합니다.
2.  **모델 및 설정:** 원하는 AI 모델과 최대 반복 횟수를 설정하고 '학습 시작'을 누릅니다.
3.  **성과 확인:** '상세 성능 분석' 탭에서 모델이 실제 데이터를 얼마나 잘 따라가는지 확인합니다.
4.  **물성 예측:** 화학 조성과 공정 조건을 입력하여 4가지 핵심 물성을 즉시 예측합니다.

# AI Materials Discovery Platform - 사용자 가이드

본 플랫폼은 전산재료 데이터와 인공지능 기술을 융합하여 신소재의 기계적 물성을 예측하고 분석하기 위해 개발되었습니다.

## 주요 기능
1. **데이터 자동 추출 및 정량화**: 엑셀 데이터를 읽어와 학습 데이터로 변환합니다.
2. **TFP 기반 확률론적 모델링**: 단순 예측뿐만 아니라 예측값의 **신뢰도(불확실성)**를 함께 산출합니다.
3. **학습용 GUI**: 데이터 학습 상황, R2, MAE 지표 및 예측 성능 그래프를 실시간으로 확인합니다.
4. **추론용 GUI**: 합금 조성 및 공정 조건을 입력하여 즉시 UTS와 항복강도를 예측합니다.

## 실행 방법
```bash
pip install -r requirements.txt  # (필요한 라이브러리들 설치)
python main.py
```

## 시스템 구성
- **Data Engine**: 엑셀 데이터의 전처리, 정규화 및 인코딩 담당
- **Model Engine**: TensorFlow Probability 기반의 심층 신경망 모델
- **Inference System**: Flask API 서버 및 PyQt6 GUI를 통한 결과 도출

## 참고 사항

- 신소재 탐색 시, 'Property Inference' 탭에서 다양한 조건(온도, 공정 등)을 입력하여 물성 변화를 시뮬레이션할 수 있습니다.

# Data Preprocessing

## 개요

이 문서는 AI Materials Discovery Platform의 데이터 전처리 구조를 설명한다.

현재 전처리는 크게 2단계로 나뉜다.

1. 1차 전처리
   - 도메인 기준 검증
   - 형식 오류 처리
   - 결측치 처리
   - 이상치 처리
2. 2차 전처리
   - 합금 지표(feature engineering) 생성

구현 위치:

- `src/engine/data_engine.py`
- `src/gui/main_window.py`

## 전처리 흐름

전처리는 `DataEngine.load_data()` 실행 시 아래 순서로 진행된다.

1. Excel 파일을 읽는다.
2. 원본 조성/공정/타깃 컬럼을 숫자형으로 변환한다.
3. 잘못된 형식 값을 집계한다.
4. 결측치 처리 전략을 적용한다.
5. 도메인 기준과 IQR 기준으로 이상치를 감지한다.
6. 이상치를 보정하거나 제거한다.
7. 전처리 결과 인덱스를 다시 정렬한다.
8. 2차 전처리가 켜져 있으면 합금 지표를 추가 생성한다.
9. 결과를 `self.df`와 `last_quality_report`에 저장한다.

## 1차 전처리

### 1. 도메인 기준 검증

도메인 검증은 단순 통계 이상치만 보는 것이 아니라, 재료공학적으로 말이 되는 범위인지 함께 확인한다.

현재 기준은 2개 부류로 나뉜다.

- 오스테나이트 조성 기준
- 고온 특성 기준

기본 범위 예시:

### 오스테나이트 조성 기준

- `Cr`: `16 ~ 26`
- `Ni`: `3.5 ~ 37`
- `Mo`: `0 ~ 5`
- `Mn`: `0 ~ 9`
- `Cu`: `0 ~ 4`
- `N`: `0 ~ 0.4`
- `C`: `0 ~ 0.15`

### 고온 특성 기준

- `Solution_treatment_temperature`: `900 ~ 1500`
- `Temperature (K)`: `273 ~ 1422`
- `Water_Quenched_after_s.t.`: `0 ~ 1`
- `Air_Quenched_after_s.t.`: `0 ~ 1`

도메인 범위는 UI의 `도메인 기준 설정`에서 직접 수정할 수 참조 수정할 수 있다.

### 2. 형식 오류 처리

수치형 컬럼은 모두 `pd.to_numeric(..., errors="coerce")`를 통해 숫자로 변환한다.

지원 방식:

- `잘못된 값을 NaN으로 변환`
- `잘못된 값이 있는 행 제거`

형식 오류 개수는 `invalid_type_cells`로 기록된다.

### 3. 결측치 처리

지원 방식:

- `평균값으로 채우기`
- `중앙값으로 채우기`
- `주변 값으로 예측(KNN)`
- `해당 행 제거`

권장 기본값:

- `중앙값으로 채우기`

### 4. 이상치 처리

이상치는 다음 두 기준을 함께 사용한다.

- 도메인 기준 범위
- IQR 기반 통계 범위

지원 방식:

- `감지 범위로 보정`
- `이상치 행 제거`
- `표시만 하고 유지`

권장 기본값:

- `감지 범위로 보정`

### 5. 이상치 민감도

IQR 기반 감지 강도를 조절하는 값이다.

- 기본값: `1.5`
- 더 엄격하게 보고 싶으면: `1.0 ~ 1.3`
- 더 느슨하게 보고 싶으면: `2.0 이상`

## 2차 전처리

2차 전처리는 1차 전처리로 정리된 데이터를 바탕으로 합금 지표를 추가 생성하는 단계다.

UI에서는 `합금 지표 생성 전처리` 체크박스로 켜고 끌 수 있다.

기본값은 `사용`이다.

### 생성되는 합금 지표

- `Cr_Ni_ratio`
- `C_plus_N`
- `Ni_eq`
- `Cr_eq`

### 계산식

- `Cr_Ni_ratio = Cr / Ni`
- `C_plus_N = C + N`
- `Ni_eq = Ni + 30*C + 0.5*Mn`
- `Cr_eq = Cr + Mo + 1.5*Si + 0.5*Nb`

### 왜 2차 전처리로 분리하는가

이 단계는 데이터 정제가 아니라 feature engineering에 가깝다.

즉 역할이 다르다.

- 1차 전처리: 데이터가 정상인지 정리
- 2차 전처리: 재료공학적으로 더 의미 있는 입력 변수 생성

따라서 순서는 아래가 가장 적절하다.

1. 도메인 검증 및 품질 처리
2. 합금 지표 생성
3. 스케일링
4. 모델 학습

## 모델 학습과의 연결

전처리 완료 후 `preprocess_data()`에서 학습용 입력과 타깃을 분리한다.

이때 입력 변수는 다음 순서로 사용된다.

1. 원본 조성/공정 변수
2. 2차 전처리로 생성된 합금 지표

그 다음 `StandardScaler`로 입력과 타깃을 각각 스케일링한다.

즉, 합금 지표 생성은 모델 내부가 아니라 전처리 단계에서 먼저 수행된다.

## 전처리 결과 확인

UI의 `데이터 전처리` 탭에서는 전처리 완료 후 결과를 표 전체로 확인할 수 있다.

특징:

- 전처리 완료 데이터를 전체 행 기준으로 출력
- 전처리 후 순서대로 다시 인덱스를 정렬
- 설정이 바뀌면 이전 결과는 무효화
- `전처리 결과 초기화` 버튼으로 결과 상태를 리셋 가능

## 전처리 리포트

전처리 후 `DataEngine.last_quality_report`에는 아래 정보가 저장된다.

- `rows_before`
- `rows_after`
- `invalid_type_cells`
- `missing_cells_before`
- `missing_cells_after`
- `outlier_cells`
- `domain_range_cells`
- `rows_removed_by_missing`
- `rows_removed_by_outlier`
- `missing_strategy`
- `outlier_strategy`
- `invalid_type_strategy`
- `feature_engineering_enabled`
- `engineered_features_added`
