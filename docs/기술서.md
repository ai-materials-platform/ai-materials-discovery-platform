# AI Materials Discovery Platform — 프로젝트 기술서

> 버전: v2.0 | 최종 작성일: 2026-03-31

---

## 1. 프로젝트 개요

### 1.1 목적

스테인리스강(오스테나이트계)의 **화학 조성 + 공정 변수 → 기계적 물성** 예측을 자동화하는 AI 플랫폼.
연구자가 Excel 데이터를 불러와 전처리·학습·추론을 데스크톱 앱 하나로 수행할 수 있도록 설계된 시스템이다.

### 1.2 예측 대상 물성 (4가지)

| 구분 | 물성 | 단위 |
|------|------|------|
| 강도(Strength) | 0.2% 항복강도 (Proof Stress) | MPa |
| 강도(Strength) | 인장강도 (UTS) | MPa |
| 연성(Ductility) | 연신율 (Elongation) | % |
| 연성(Ductility) | 단면 수축률 (Area Reduction) | % |

### 1.3 대상 소재

- **소재군**: 오스테나이트계 스테인리스강
- **참조 데이터**: `STMECH_AUS_SS.xls` (SSINA 오스테나이트 조성표 기반)

## 2. 시스템 아키텍처

### 2.1 전체 구조

```
사용자
  └→ Electron 데스크톱 앱 (HTML/CSS/JS)
       └→ HTTP REST + SSE (localhost:5001)
            └→ Flask API Server (Python 백엔드)
                 ├→ DataEngine  (전처리 / Feature Engineering / 스케일링)
                 └→ ModelEngine (RF / GBM / MLP / TFP 앙상블)
```

- **Electron 메인 프로세스**가 앱 실행 시 Python Flask 서버를 자식 프로세스(subprocess)로 기동
- Flask 준비 완료 전까지 **스플래시 스크린** 표시 (단계별 상태 텍스트 + 프로그레스 바)
- 준비 완료 후 Electron 렌더러 ↔ Flask API 간 REST + SSE 통신

### 2.2 레이어별 역할

| 레이어 | 위치 | 역할 |
|--------|------|------|
| 스플래시 | `electron/splash.html` | 앱 기동 중 로딩 화면 표시 |
| Electron 메인 | `electron/main.js` | Python 프로세스 관리, 창 생성/제어 |
| Electron 렌더러 | `electron/renderer/` | 탭 UI, API 호출, 차트 렌더링 |
| Flask API | `python/main_api.py` + `api/routes_*.py` | REST 엔드포인트, SSE 스트림 |
| Data Engine | `python/engine/data_engine.py` | 전처리, Feature Engineering, 스케일링 |
| Model Engine | `python/engine/model_engine.py` | 모델 생성, 학습, 예측, 저장/로드 |
| Config | `python/config/settings.py` | 전역 상수, 경로, 도메인 기준 중앙 관리 |

### 2.3 프로세스 기동 흐름

```
앱 실행
  → [1] Electron 메인 프로세스 시작
  → [2] 스플래시 창 즉시 표시
  → [3] Python Flask 서버 subprocess 기동
  → [4] GET /health 폴링 (0.5초 간격, 최대 30회)
  → [5] 응답 확인 → 스플래시 페이드 아웃
  → [6] 메인 창 페이드 인
  → [7] 앱 종료 시 Python 프로세스 함께 종료
```

#### 지원 모델 4종

| 모델 ID | 이름 | 구현 방식 | 불확실성 산출 방식 |

| `TFP` | 확률론적 앙상블 | Bootstrap 5-MLP Ensemble | 앙상블 표준편차 (실측) |

## 5. 데이터 파이프라인

### 5.1 입력 데이터 구조

- **파일 형식**: `.xls` (Excel 97-2003) 및 `.xlsx` / `.xlsm` (Excel 2007+) — `pd.read_excel()`이 확장자에 따라 엔진 자동 선택
- **원본 입력 변수 (29개)**

| 분류 | 변수 |
|------|------|
| 합금 원소 (조성) | Cr, Ni, Mo, Mn, Si, Nb, Ti, Zr, Ta, V, W, Cu, N, C, B, P, S, Co, Al, Sn, Pb |
| 공정 변수 | Solution_treatment_temperature, Solution_treatment_time(s), Water_Quenched_after_s.t., Air_Quenched_after_s.t. |
| 미세조직 | Grains mm⁻², Type of melting, Size of ingot, Product form |
| 시험 조건 | Temperature (K) |

### 5.2 1차 전처리 (데이터 정제)

```
Excel 로드
  → 수치형 변환 (pd.to_numeric, errors='coerce')
  → 형식 오류 집계 (invalid_type_cells)
  → 결측치 처리 (평균 / 중앙값 / KNN / 행 제거)
  → 도메인 기준 검증 (오스테나이트 범위 + 고온 특성 범위)
  → 이상치 처리 (IQR 기반 + 도메인 기준 / clip / 제거 / 유지)
  → 인덱스 재정렬
```

#### 도메인 기준값 (기본)

| 그룹 | 변수 | 허용 범위 |
|------|------|----------|
| 오스테나이트 조성 | Cr | 16 ~ 26 |
| 오스테나이트 조성 | Ni | 3.5 ~ 37 |
| 오스테나이트 조성 | Mo | 0 ~ 5 |
| 오스테나이트 조성 | Mn | 0 ~ 9 |
| 오스테나이트 조성 | Cu | 0 ~ 4 |
| 오스테나이트 조성 | N | 0 ~ 0.4 |
| 오스테나이트 조성 | C | 0 ~ 0.15 |
| 고온 특성 | Solution_treatment_temperature | 900 ~ 1500 |
| 고온 특성 | Temperature (K) | 273 ~ 1422 |
| 고온 특성 | Water_Quenched_after_s.t. | 0 ~ 1 |
| 고온 특성 | Air_Quenched_after_s.t. | 0 ~ 1 |

### 5.3 2차 전처리 (Feature Engineering)

| 변수명 | 계산식 | 의미 |
|--------|--------|------|
| `Cr_Ni_ratio` | `Cr / Ni` | 크롬-니켈 비율 (상 안정성 지표) |
| `C_plus_N` | `C + N` | 침입형 원소 합 (강화 효과) |
| `Ni_eq` | `Ni + 30×C + 0.5×Mn` | 니켈 당량 (오스테나이트 안정화) |
| `Cr_eq` | `Cr + Mo + 1.5×Si + 0.5×Nb` | 크롬 당량 (페라이트 안정화) |

### 5.4 피처 컬럼 선택 (학습 전)

전처리 완료 후, 학습 전에 사용자가 입력 피처를 개별적으로 선택/해제할 수 있다.

- `DataEngine.selected_feature_cols` (기본값 `None` = 전체 사용)
- `set_selected_features(cols)` 로 명시적 지정, `get_available_feature_cols()` 로 선택 가능 목록 조회
- 설정된 선택은 `models/data_engine.pkl` 직렬화에 포함되므로, 추론 시에도 학습 시와 동일한 컬럼 집합이 자동 유지됨
- 관련 API: `GET /features/available`, `POST /features/select`, `GET /features/selected`

### 5.5 스케일링 및 분할

```
선택된 입력 피처 (최대 33개, 사용자 선택에 따라 가변)
  → StandardScaler (X, y 각각 별도 스케일러)
  → Train/Test split (기본 80:20)
  → 모델 학습
```

### 5.6 전처리 리포트 항목

`DataEngine.last_quality_report`에 저장:

```
rows_before / rows_after
invalid_type_cells
missing_cells_before / missing_cells_after
outlier_cells / domain_range_cells
rows_removed_by_missing / rows_removed_by_outlier
missing_strategy / outlier_strategy / invalid_type_strategy
feature_engineering_enabled / engineered_features_added
```

## 6. API 명세

### 6.1 전체 엔드포인트

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/health` | GET | 서버 상태 확인 (스플래시 Health Check용) |
| `/preprocess/load` | POST | Excel 파일 로드 |
| `/preprocess/run` | POST | 전처리 실행 (품질 옵션 포함) |
| `/preprocess/preview` | GET | 전처리 결과 미리보기 (JSON) |
| `/preprocess/domain-ranges` | GET/PUT | 도메인 기준 조회/수정 |
| `/features/available` | GET | 전처리 후 선택 가능한 피처 컬럼 목록 반환 |
| `/features/select` | POST | 학습에 사용할 피처 컬럼 지정 `{ "cols": [...] }` |
| `/features/selected` | GET | 현재 선택된 피처 컬럼 목록 반환 |
| `/train/start` | POST | 모델 학습 시작 |
| `/train/status` | GET | 학습 진행 상태 (SSE, `text/event-stream`) |
| `/performance/metrics` | GET | R², MAE 등 성능 지표 |
| `/performance/parity` | GET | Parity Plot 데이터 |
| `/performance/heatmap` | GET | 상관관계 히트맵 데이터 |
| `/inference/predict` | POST | 조성 입력 → 4개 물성 예측 |

### 6.2 추론 요청/응답 예시

**요청** (`POST /inference/predict`):
```json
{
  "Cr": 18.0, "Ni": 8.0, "Mo": 0.0, "Mn": 2.0,
  "Si": 1.0, "C": 0.08, "N": 0.05,
  "Solution_treatment_temperature": 1050,
  "Temperature (K)": 293
}
```

**응답**:
```json
{
  "proof_stress":   { "mean": 250.3, "std": 12.1 },
  "uts":            { "mean": 580.7, "std": 18.4 },
  "elongation":     { "mean": 45.2,  "std": 3.1  },
  "area_reduction": { "mean": 62.0,  "std": 5.7  }
}
```
